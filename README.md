# life-record

The Life-OS bot's memory — Rahul's durable record. Human-readable by design.

- `context/long-term.md` — deep memory: identity, goals, learned patterns (bot proposes updates, Rahul confirms)
- `context/short-term.md` — working memory: this week's focus, active threads (updated live, tidied nightly)
- `log/` — raw timestamped messages, one file per day (source of truth)
- `days/` `weeks/` `months/` — consolidated digests (the compressed memory)
- `data/` — rolling numbers: nutrition, training, recovery, growth, projects
- `media/` — photos, PDFs, voice notes (Git LFS)
